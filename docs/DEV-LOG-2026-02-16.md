## Technical Decisions

### Animation Strategy
**Initial Approach**: Heavy spring animations with physics
**Final Approach**: Simple CSS transitions with minimal springs
**Reason**: Better performance, no runtime errors, smoother 60fps

### Color System
**Initial Approach**: Rainbow/multicolor (5+ colors)
**Final Approach**: Strict 3-color (Green, Blue, Purple)
**Reason**: Cleaner design, professional look, easier maintenance

### Navigation
**Initial Approach**: Floating glass navbar
**Final Approach**: Static navbar at top
**Reason**: Better UX, more space for content, cleaner mobile experience

## Challenges Faced

1. **Spring Animation Errors**
   - Error: "Only two keyframes supported with spring"
   - Solution: Replaced complex spring arrays with simple transitions

2. **Responsive Design Complexity**
   - Challenge: Making heavy animations work on mobile
   - Solution: Conditional rendering based on screen size

3. **Color Consistency**
   - Challenge: Too many colors looked chaotic
   - Solution: Strict 3-color palette enforced

4. **Performance on Mobile**
   - Challenge: Battery drain from continuous animations
   - Solution: Reduced/removed background animations on mobile
